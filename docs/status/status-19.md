## 概要

1D過渡電熱 FDM ソルバー (`scripts/fdm_gs_1d_v3.py`) に3つの機能追加・修正を実施。

## 変更内容

### 1. 断熱境界条件バグ修正

**問題**: `T_left=None` / `T_right=None`（断熱指定）のとき、境界コンダクタンス `C_left_bnd` が
対角係数 `a_P` に加算されるが、右辺 `b_bnd` は0のまま。結果として端部セルの温度が0K方向に
引っ張られ、均一発熱でも端部だけ温度が低い不正な解になっていた。

**修正**: `a_P_bnd` と `b_bnd` の境界コンダクタンス加算を `T_left is not None` / `T_right is not None`
の場合のみに限定。断熱時はどちらも0（ゼロ勾配）。

**検証**: 断熱 + 均一ジュール加熱で全セル同一温度を確認。

### 2. 輻射ソース項の実装

**物理モデル**:
```
q_rad = ε · σ · A_rad · (T⁴ - T_env⁴)  [W/cell]
```

**線形化手法**: 因数分解による等価熱伝達係数
```
h_rad = ε · σ · (T*² + T_env²) · (T* + T_env)  [W/(m²·K)]
```
これにより `q_rad ≈ h_rad · A_rad · (T - T_env)` と線形形式に落ちる。
外側反復で `T*` を更新し、非線形性を解消。

**追加パラメータ**:
- `seg_epsilon`: 区間ごとの輻射率 [-]（デフォルト0.0＝輻射なし）
- `seg_areas_rad`: 区間ごとの輻射面積 [m²]（デフォルト＝放熱面積と同じ）

**検証**: Demo 2 で定常解のエネルギー収支 balance_err = O(1e-12) W を確認。
解析値との一致も検証済み。

### 3. LineArea リストヘルパー関数

`solve_from_line_areas(segments: list[LineArea], ...)` を追加。
LineArea dataclass のリストを直列接続としてソルバーに渡すラッパー。

**主な仕様**:
- 電流は先頭セグメントの値を使用（直列＝全区間同一電流）
- LineArea の `T_env` は℃定義 → K に自動変換
- 輻射面積 `A_rad` が None の場合は放熱面積 `A1` を使用
- 各区間のセル分割数は `LineArea.n_cells` で指定

**追加フィールド**:
- `LineArea.A_rad`: 輻射面積 [m²]（オプション）
- `LineArea.n_cells`: セル分割数（デフォルト10）

## 設計上の懸念・メモ

- 輻射の線形化 `h_rad = ε·σ·(T²+T_env²)·(T+T_env)` は `a²-b²=(a+b)(a-b)` の因数分解に基づく厳密な等価変換。外側反復で `T*` が収束すれば非線形性は完全に解消される。
- Demo 1 の balance_err が大きいのは過渡解析のため（定常に達していない）。蓄熱項を考慮すれば零。
- 断熱BCバグは status-18 の検算スクリプトでは発覚しなかった（温度固定BCのみテスト）。今回 `T_left=None` を試して初めて判明。
