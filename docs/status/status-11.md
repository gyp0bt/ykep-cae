### 分析と知見

#### 1. 定常SIMPLE vs 非定常の安定性

- **定常SIMPLEはq_vol付き自然対流問題で発散する**（全27ケース発散）
- 原因: q_volが連続的にエネルギーを投入するため、速度場が追いつく前に温度が発散
- **非定常time-marchingは安定**（全24ケースNaN発散なし）
- 教訓: 体積発熱源の自然対流問題では非定常解析が必須

#### 2. 修正Rayleigh数 Ra* と対流発達

- Ra* = gβqL⁵/(kνα) は L に対して5乗で増大
- Ra* < 10³: 対流微弱、伝導支配（Phase 1）
- Ra* ~ 10⁵: 自然対流が発達し始める（Phase 2, L=0.05）
- Ra* > 10⁶: 強い対流、100ステップでは平衡未到達（Phase 2, L=0.1）

#### 3. 境界条件の影響

- **密閉(A)**: 温度が境界壁で固定されるため、ΔT_maxが最も抑制される
  - L=0.05, q=100: ΔT=73K（最小）
- **半開放(B)・3辺開放(C)**: 
  - 煙突効果で流速が5-8倍に増大（v_max: 0.05→0.25-0.37 m/s）
  - ΔT_maxは密閉の5-7倍に増大（506-537K vs 73K）
  - OUTLET_PRESSURE境界からの流出・流入が支配的
- **開放BCで小さい領域サイズで表現できるか**: 
  - 開放BCは流入冷気と流出温気を表現でき物理的により現実的
  - しかし現在のOUTLET_PRESSURE（ゼロ勾配外挿）は単純すぎ、大きな温度差が生じる
  - 密閉の方が数値的に安定で、物理的に妥当な温度分布を得やすい

#### 4. ドメインサイズの設計指針

- **L/d_heater比**: 発熱体サイズに対するドメインサイズの比率が重要
  - 調査では d_heater ≈ L×0.2 → L/d_heater = 5
  - Ra*がL⁵に比例するため、ドメインが2倍になるとRa*は32倍
- **推奨**: 
  - 空気の自然対流: L ≤ 0.05m（Ra* ~ 10⁵）が現ソルバーで安定的に解ける範囲
  - L=0.1m以上: より多くのタイムステップ（500-1000）、より小さいdtが必要
  - メッシュ解像度: nx=ny=20で定性的傾向は把握可能、定量的精度にはnx=40以上が望ましい

#### 5. オープン境界について

- 現在のOUTLET_PRESSURE実装は基本的なゼロ勾配外挿のみ
- **改善提案**:
  - 特性ベース（characteristic-based）の非反射境界条件
  - 対流流出境界（convective outflow BC）
  - これらにより、より小さいドメインでの安定した解析が可能になる
- 当面は**密閉キャビティ + 十分なドメインサイズ**が最も信頼性が高い

### 汎用的な設計指針まとめ

| 項目 | 推奨 |
|------|------|
| 解析モード | 非定常（q_vol付きでは必須） |
| ドメインサイズ | 発熱体の5-10倍（L/d ≥ 5） |
| メッシュ | 20×20（定性的）、40×40（定量的） |
| 境界条件 | 初期調査は密閉が安定。開放BCは改良後に推奨 |
| 時間刻み | dt < dx²/(2α)（拡散安定性） |
| タイムステップ数 | Ra*<10⁵: 100-200、Ra*>10⁶: 500+ |
| SIMPLE緩和 | α_u=0.5, α_p=0.2, α_T=0.8（非定常時） |
| Ra*制限 | 現ソルバーで安定: Ra* < 10⁶ |

## 設計上の懸念

- **OUTLET_PRESSUREの限界**: 現在のゼロ勾配外挿は自然対流の流出境界として不十分。
  温気が境界セルに滞留し、非物理的な温度分布を生む。特性ベースの境界条件が必要。
- **圧力基準点**: OUTLET_PRESSURE使用時、最初の流体セルの圧力固定が適切でない可能性。
  OUTLET_PRESSURE面の圧力を基準にすべき。
- **非定常SIMPLEの収束**: 各タイムステップでのSIMPLE反復数（20-30）が十分でない場合、
  累積誤差が蓄積する。SIMPLEC方式への拡張が望ましい。
- **空気実物性でのRa*爆発**: Ra*がL⁵に比例するため、実スケール（L=0.1m+）の問題は
  乱流モデル（Phase 5）なしでは解けない可能性が高い。
