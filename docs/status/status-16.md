# status-16: AMG圧力ソルバー + 面ベース質量残差 + 適応的緩和

[<- status-index](status-index.md) | [<- README](../../README.md)

## 日付

2026-04-02

## 概要

status-15 の TODO（AMG 前処理・適応的緩和）を消化。
加えて、質量残差の計算方法に根本的なバグを発見・修正し、
2cm 領域 20×20 セルの空気実物性自然対流が全手法で収束するようになった。

## 実施内容

### 1. 圧力方程式への AMG 前処理適用

`_solve_pressure_amg()` を新設。PyAMG の Ruge-Stuben AMG を前処理として
BiCGSTAB に組み合わせ、ラプラシアン型の圧力補正方程式を高速に解く。

- `_PressureAMGCache` クラスでスパースパターン不変時に AMG 階層を再利用
- `NaturalConvectionInput.pressure_solver = "amg"` で有効化
- `max_pressure_iter` パラメータで圧力方程式の反復数を個別制御可能

### 2. 面ベース質量残差への修正（根本的バグ修正）

**発見した問題**: 従来の `_compute_mass_residual()` はセル中心速度の中心差分で
発散を計算していたが、圧力補正方程式は Rhie-Chow 面速度ベースの発散を使用。
この不整合により：

- 圧力補正が面速度の発散を減少させても、セル中心の発散は減少しない
- SIMPLE 反復の mass 残差が収束しない
- 全手法（SIMPLE/SIMPLEC/PISO）で同じ問題が発生

**修正**: `compute_face_mass_residual()` を assembly.py に新設し、
Rhie-Chow 面速度と同じ面ベース発散で質量残差を計算するよう変更。

### 3. 初期残差方式への変更

残差の計算を「最終残差」から「初期残差」（OpenFOAM 方式）に変更。

- 従来: `||b - A*x_new|| / ||b||` → 線形ソルバーの精度を測定（常に小さい）
- 修正: `||b - A*x_old|| / ||b||` → 外部反復の収束を正しく測定

### 4. 適応的緩和係数

`_adapt_relaxation()` メソッドを追加。残差の減少率に応じて：
- 残差減少時 (ratio < 0.8): alpha_u ×1.1 (max 0.9), alpha_p ×1.1 (max 0.5)
- 残差増大時 (ratio > 1.2): alpha_u ×0.8 (min 0.1), alpha_p ×0.8 (min 0.05)

`NaturalConvectionInput.adaptive_relaxation = True` で有効化。

## 開発運用メモ

- 面ベース質量残差への修正は「STA2防止」の観点で重要な発見。
  従来のセル中心ベース残差では、圧力補正の効果を正しく評価できていなかった。
  これにより「収束しない」と判定されていた空気実物性ケースが実は正常に動作していた。
- 初期残差方式への変更も同様に重要。線形ソルバーの最終残差は常に小さく、
  外部反復の収束を正しく反映しない。OpenFOAM と同じ初期残差方式にすることで
  定常解析の収束判定が適切になった。
- PyAMG はオプション依存（`pip install xkep-cae-fluid[amg]`）。
  テストでは `pytest.importorskip("pyamg")` で自動スキップ。
- ベンチマークスクリプト（bench_amg*.py）は tee でログ保存済み。
